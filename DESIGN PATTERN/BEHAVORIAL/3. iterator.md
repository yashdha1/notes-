**Idea:** Provide a way to **sequentially access elements** of a collection without exposing its underlying structure (list, tree, stack, etc.). The collection doesn't need to know how it's being traversed.
### Structure
- **Iterator (Abstract):** Interface with `__next__()` and `__iter__()` methods
- **Concrete Iterator:** Implements traversal logic, tracks current position
- **Iterable (Collection):** The object that returns an iterator
- **Client:** Uses the iterator to traverse without knowing internals

### Python Example — Book Collection with Multiple Iterators

```python
from abc import ABC, abstractmethod

# ── Abstract Iterator ─────────────────────────────────────
class BookIterator(ABC):
    @abstractmethod
    def __iter__(self):
        pass
    @abstractmethod
    def __next__(self):
        pass

# ── The Object / Data Model ───────────────────────────────
class Book:
    def __init__(self, title, genre, pages):
        self.title = title
        self.genre = genre
        self.pages = pages
    def __repr__(self):
        return f"'{self.title}' [{self.genre}, {self.pages}pg]"


# ── Concrete Iterator 1: Traverse All Books ───────────────
class AllBooksIterator(BookIterator):
    def __init__(self, books):
        self._books = books
        self._index = 0
    def __iter__(self):
        return self
    def __next__(self):
        if self._index < len(self._books):
            book = self._books[self._index]
            self._index += 1
            return book
        raise StopIteration


# ── Concrete Iterator 2: Filter by Genre ─────────────────
class GenreIterator(BookIterator):
    def __init__(self, books, genre):
        self._books = [b for b in books if b.genre == genre]
        self._index = 0
    def __iter__(self):
        return self
    def __next__(self):
        if self._index < len(self._books):
            book = self._books[self._index]
            self._index += 1
            return book
        raise StopIteration


# ── Concrete Iterator 3: Sorted by Page Count ────────────
class PageSortedIterator(BookIterator):
    def __init__(self, books):
        self._books = sorted(books, key=lambda b: b.pages)
        self._index = 0
    def __iter__(self):
        return self
    def __next__(self):
        if self._index < len(self._books):
            book = self._books[self._index]
            self._index += 1
            return book
        raise StopIteration

# ── Collection (Iterable) ─────────────────────────────────
class BookShelf:
    def __init__(self):
        self._books = []
    def add_book(self, book):
        self._books.append(book)
    def all_books(self):
        return AllBooksIterator(self._books)
    def by_genre(self, genre):
        return GenreIterator(self._books, genre)
    def by_pages(self):
        return PageSortedIterator(self._books)


# ── Client ────────────────────────────────────────────────
shelf = BookShelf()
shelf.add_book(Book("Dune",             "Sci-Fi",  412))
shelf.add_book(Book("1984",             "Dystopia", 328))
shelf.add_book(Book("Foundation",       "Sci-Fi",  244))
shelf.add_book(Book("Brave New World",  "Dystopia", 311))
shelf.add_book(Book("Hyperion",         "Sci-Fi",  482))

print("=== All Books ===")
for book in shelf.all_books():
    print(book)

print("\n=== Sci-Fi Only ===")
for book in shelf.by_genre("Sci-Fi"):
    print(book)

print("\n=== Sorted by Page Count ===")
for book in shelf.by_pages():
    print(book)
```

```
'1984' [Dystopia, 328pg]
'Foundation' [Sci-Fi, 244pg]
'Brave New World' [Dystopia, 311pg]

=== Sci-Fi Only ===

'Dune' [Sci-Fi, 412pg]
'Hyperion' [Sci-Fi, 482pg]
```

### Python's Built-in Iterator Protocol
Python has **native support** for this pattern through dunder methods:

```python
class CountDown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):   
        if self.current <= 0:
            raise StopIteration
        val = self.current
        self.current -= 1
        return val

for n in CountDown(5):
    print(n)    
```

> In Python, any object implementing `__iter__` and `__next__` is automatically compatible with `for` loops, `list()`, `zip()`, `enumerate()`, etc.

### How Data Flows

```
Client
  │
  └──► Collection.get_iterator()
              │
              └──► Iterator (tracks position)
                        │
                        ├── __next__() ──► returns next element
                        └── StopIteration ──► traversal ends
```

---

### When to Use
- You want to traverse a collection **without exposing its internals**
- You need **multiple traversal strategies** on the same collection
- You want a **uniform interface** across different collection types
- When dealing with **lazy evaluation** — generate items on demand

### Real-World Examples
- **Python's `for` loop** — uses iterator protocol under the hood
- **Database cursors** — fetch rows one by one from a query result
- **File reading** — `for line in file` reads line by line lazily
- **Pagination APIs** — fetch one page at a time from REST APIs
- **`itertools` module** — `chain`, `islice`, `cycle` are all iterators

### Pros & Cons

|Pros|Cons|
|---|---|
|Hides internal collection structure|Extra classes for simple collections|
|Multiple iterators can run simultaneously|Stateful — can be tricky to manage|
|Supports lazy/on-demand evaluation|One-directional by default|
|Clean separation of traversal logic|Bidirectional traversal needs extra work|

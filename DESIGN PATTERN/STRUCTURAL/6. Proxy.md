Issue: 
**Proxy** is a structural design pattern that lets you provide a substitute or placeholder for another object. A proxy controls access to the original object, allowing you to perform something either before or after the request gets through to the original object. 

You could implement lazy initialization: create this object only when it’s actually needed. All of the object’s clients would need to execute some deferred initialization code. Unfortunately, this would probably cause a lot of code duplication.

In an ideal world, we’d want to put this code directly into our object’s class, but that isn’t always possible. For instance, the class may be part of a closed 3rd-party library.

sol : 
The Proxy pattern suggests that you create a new proxy class with the same interface as an original service object. Then you update your app so that it passes the proxy object to all of the original object’s clients. Upon receiving a request from a client, the proxy creates a real service object and delegates all the work to it. 
But what’s the benefit? If you need to execute something either before or after the primary logic of the class, the proxy lets you do this without changing that class. Since the proxy implements the same interface as the original class, it can be passed to any client that expects a real service object. 


The **Proxy** pattern provides a **substitute or placeholder** for another object to control access to it.
### Concept
> "A proxy controls access to the real object — adding a layer in between."

### Types of Proxies :

|Type|Purpose|Example|
|---|---|---|
|**Virtual**|Lazy initialization, delay costly creation|Load image only when needed|
|**Protection**|Access control / permissions|Admin-only resources|
|**Remote**|Represents object in another location|API / network calls|
|**Caching**|Cache results of expensive operations|Cache DB query results|

### Example -

```python
from abc import ABC, abstractmethod
import time

# --- SUBJECT - common interface ---
class Database(ABC):
    @abstractmethod
    def query(self, sql: str) -> str:
        pass
# --- REAL SUBJECT - the actual object ---
class RealDatabase(Database):
    def __init__(self):
        print("  [DB] Connecting to database...")
        time.sleep(0.5)  # simulate costly connection

    def query(self, sql: str) -> str:
        print(f"  [DB] Executing: {sql}")
        return f"Results for: '{sql}'"


# ============================================
# 1. VIRTUAL PROXY - lazy initialization
# ============================================
class LazyDatabaseProxy(Database):
    def __init__(self):
        self._real_db = None          # not created yet!

    def query(self, sql: str) -> str:
        if self._real_db is None:
            print("  [Lazy Proxy] First access — initializing DB...")
            self._real_db = RealDatabase()   # created only when needed
        return self._real_db.query(sql)


# ============================================
# 2. PROTECTION PROXY - access control
# ============================================
class ProtectedDatabaseProxy(Database):
    _ALLOWED_ROLES = {"admin", "developer"}

    def __init__(self, user_role: str):
        self._role    = user_role
        self._real_db = RealDatabase()

    def query(self, sql: str) -> str:
        if self._role not in self._ALLOWED_ROLES:
            raise PermissionError(
                f"  [Protection Proxy] ❌ Role '{self._role}' is not allowed!"
            )
        print(f"  [Protection Proxy] ✅ Access granted for role: {self._role}")
        return self._real_db.query(sql)


# ============================================
# 3. CACHING PROXY - cache expensive results
# ============================================
class CachingDatabaseProxy(Database):
    def __init__(self):
        self._real_db = RealDatabase()
        self._cache: dict[str, str] = {}

    def query(self, sql: str) -> str:
        if sql in self._cache:
            print(f"  [Cache Proxy] ✅ Cache HIT  → '{sql}'")
            return self._cache[sql]

        print(f"  [Cache Proxy] ❌ Cache MISS → forwarding to DB")
        result = self._real_db.query(sql)
        self._cache[sql] = result
        return result


# --- Usage ---
print("=" * 50)
print("1️⃣  VIRTUAL PROXY (Lazy Init)")
print("=" * 50)
lazy_db = LazyDatabaseProxy()          # no DB connection yet
print("  Proxy created — DB not connected yet!")
print(lazy_db.query("SELECT * FROM users"))   # connects NOW
print(lazy_db.query("SELECT * FROM orders"))  # reuses connection

print("\n" + "=" * 50)
print("2️⃣  PROTECTION PROXY")
print("=" * 50)
admin_db = ProtectedDatabaseProxy("admin")
print(admin_db.query("SELECT * FROM secrets"))

guest_db = ProtectedDatabaseProxy("guest")
try:
    print(guest_db.query("SELECT * FROM secrets"))
except PermissionError as e:
    print(e)

print("\n" + "=" * 50)
print("3️⃣  CACHING PROXY")
print("=" * 50)
cached_db = CachingDatabaseProxy()
print(cached_db.query("SELECT * FROM products"))  # miss
print(cached_db.query("SELECT * FROM products"))  # hit!
print(cached_db.query("SELECT * FROM users"))     # miss
print(cached_db.query("SELECT * FROM products"))  # hit!
```

### Key Roles

|Role|In Example|
|---|---|
|**Subject**|`Database` (common interface)|
|**Real Subject**|`RealDatabase` (actual logic)|
|**Proxy**|`LazyProxy`, `ProtectedProxy`, `CachingProxy`|
|**Client**|calls proxy exactly like real object|

### Proxy vs Decorator
||Proxy|Decorator|
|---|---|---|
|**Purpose**|Control access|Add behavior|
|**Knows real object?**|Manages it internally|Receives it externally|
|**Focus**|Access & lifecycle|Feature stacking|
### When to Use
- **Lazy loading** of heavy resources
- **Access control** and permission checks
- **Caching** expensive operations
- **Logging, monitoring** without touching real object

Pydantic is a **Python library for data validation and settings management using Python type hints**. In simpler terms, it helps you ensure that the data your program receives or works with is of the correct type and format, and it makes working with structured data much cleaner and safer.

Pydantic uses **Python type annotations** to define the shape of data. Then, it **validates** that any data matches those types, automatically converting compatible types when possible.

```python
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name: str
    email: str

# Create a user instance
user = User(id='123', name='Alice', email='alice@example.com')
print(user.id, type(user.id))
```

**Deserialization** = taking external data and creating a validated Python object and deserialization, loading data into a Pydantic model.
1. parse the input data to the correct datatype. conversion should be valid ex: integer string -> int is valid but vica versa is not true. 
2. run the default validators or run the **@validators** custom defined.

**Serialization** = converting a Pydantic model back into a format suitable for storage or transmission.
- `user.dict()` → Python dictionary
- `user.json()` → JSON string
```
	- `dict()` loops over all fields in the model.
	- Converts nested models recursively.
	- Applies field aliases if defined.
	- Skips default/optional fields if specified.
	- `json()` calls `json.dumps()` on the dict representation.
```

```python
from fastapi import FastAPI
from pydantic import BaseModel

class Root(BaseModel):
    name : str
    port: int
    
path = Root(name="fastapi", port=8000)
app = FastAPI()

@app.post('/')
async def root(req : Root): # only accepts the requsts in the form of the Root:
    name = req.name
    port = req.port
    return f"processed {name} with the port {port}"
```

req data is in the form of the xml or http format : 
#### 1. **Deserialization**
converting **external data** (usually JSON from HTTP request) into **Python objects**.
In your code:
```python
	@app.post('/')  
	async def root(path: Root):  
	    return path
```

- `path: Root` tells FastAPI to expect a JSON body that fits the `Root` model.
- Suppose client sends:

{  
  "name": "fastapi",  
  "port": "8000"  
}
- steps : FastAPI does **deserialization**:
1. Receives JSON as string.
2. Converts JSON → Python dict.
dict -> {"name": "fastapi", "port": "8000"}
3. Passes dict to Pydantic:
path = Root(**data)
4. **Pydantic parses the dict**:
    - `"name": "fastapi"` → `str` 
    - `"port": "8000"` → `int` 
5. Creates a **fully typed Python object**:
Root(name="fastapi", port=8000)
 This is **deserialization + validation**.
`req: Root` tells FastAPI to expect a JSON body that fits the `Root` model.

#### 2. **Serialization**

**Serialization** = converting **Python objects** into **external format** (JSON, dict) to send over the network.
When you return `path`:
return path
FastAPI internally does **serialization**:

1. Detects `path` is a **Pydantic model**.
    
2. Calls `path.dict()` or `path.json()` internally.
    
3. Converts it to JSON for the HTTP response:
    

{  
  "name": "fastapi",  
  "port": 8000  
}

- This is now a JSON string that the client can understand.


##### pythonic cycles on the deserialisation and serialisation. 
- **Deserialization = Input parsing + validation**  
    (`JSON → dict → Root model`)
- **Serialization = Preparing response**  
    (`Root model → dict → JSON`)


### params : 

query params-> 
path params-> 
Pydantic model body params-> 

- If the parameter is also declared in the **path**, it will be used as a path parameter.
- If the parameter is of a **singular type** (like `int`, `float`, `str`, `bool`, etc) it will be interpreted as a **query** parameter.
- If the parameter is declared to be of the type of a **Pydantic model**, it will be interpreted as a request **body**. 


* *Annotated* : for the custome logic in the query params : Query parameters :

### conversions :

- convert *json -> dict*                      **json.loads(text)** 
- convert *dict -> pydantic*               **ModelName(**dict)**  
- convert *pydantic -> dict*               **ModelName.model_dump()** 
- convert *dict -> json*                      **json.dumps(dict)** 
  
- *json -> pydantic* directly         **ModelName.model_validate_json(json_data)**
- *pydantic -> json* directly         **ModelName.model_dump_json()**
